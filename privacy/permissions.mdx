---
title: "Chrome permissions used by Save Sora"
description: "A plain-language explanation of every Chrome permission and host permission Save Sora requests, and exactly why each one is needed."
---

Save Sora requests only the permissions required for its single purpose: letting you review and selectively download your own Sora videos and drafts. This page explains what each permission does and why the extension needs it.

<Note>
  All permissions are narrowly scoped to backing up your own Sora videos. No permission is used for advertising, analytics, or any purpose unrelated to downloading your own files.
</Note>

## Chrome API permissions

These permissions give the extension access to specific Chrome browser APIs.

| Permission | Why it's needed |
|---|---|
| `alarms` | Schedules a periodic check for new Save Sora releases on GitHub every 30 minutes so you are notified when an update is available |
| `downloads` | Saves the video files and CSV exports you select to your device using Chrome's built-in download API |
| `offscreen` | Creates a hidden extension document used only to assemble the ZIP archive in the browser and generate a local download URL before the finished file is saved |
| `power` | Keeps your system awake during long downloads so a sleep or screen-off event does not interrupt the transfer mid-way |
| `scripting` | Runs packaged extension code inside the Sora tab to read your published videos and drafts from your existing logged-in session — the injected code ships inside the extension, it is not fetched remotely |
| `storage` | Saves local-only extension settings and UI state (theme, sort order, default source, renamed titles, selection state, removed state, downloaded state) using `chrome.storage.local` |
| `tabs` | Opens and manages the inactive background Sora tab the extension uses to load Sora, wait for the page to be ready, and fetch your video list from your logged-in session |
| `unlimitedStorage` | Allows the IndexedDB resumable backup cache to grow beyond Chrome's default storage quota when you have a large library |

<AccordionGroup>
  <Accordion title="Why does Save Sora need scripting permission?">
    The `scripting` permission lets the extension inject its own packaged JavaScript into the Sora tab. This is the only way for the extension to read your Sora library from your existing logged-in session without requiring you to paste tokens or cookies manually.

    The injected code is bundled inside the extension package — it is not downloaded from a remote server at runtime. You can inspect it in `background.js` in the [open-source repository](https://github.com/alpha1337/save-sora).
  </Accordion>
  <Accordion title="Why does Save Sora need tabs permission?">
    The `tabs` permission allows the extension to open a single inactive Sora tab in the background, monitor when that tab has finished loading, and close or reuse it between fetches. Without this, the extension cannot load Sora in the background and wait for your library to be available.
  </Accordion>
  <Accordion title="Why does Save Sora need offscreen permission?">
    Chrome's Manifest V3 service workers cannot create Blob URLs directly. The `offscreen` permission lets the extension open a hidden, local-only extension page solely to assemble the ZIP archive from the selected video streams and produce the final download URL. The offscreen document is created only during a ZIP download and does not interact with any external network.
  </Accordion>
  <Accordion title="Why does Save Sora need unlimitedStorage?">
    Chrome imposes a default storage quota on extensions using IndexedDB. If your Sora library is large, the resumable backup cache — which lets downloads recover after an interruption — can exceed that default quota. The `unlimitedStorage` permission removes that cap so large libraries can be handled without errors.
  </Accordion>
</AccordionGroup>

## Host permissions

These permissions grant the extension network access to specific domains. Access is limited to the domains required for the extension's single purpose.

| Host | Why it's needed |
|---|---|
| `https://sora.chatgpt.com/*` | Access your Sora published videos and drafts from your logged-in session |
| `https://videos.openai.com/*` | Download and preview the Sora video files you select |
| `https://ogimg.chatgpt.com/*` | Display video thumbnail previews in the review list |
| `https://api.github.com/*` | Check for new Save Sora releases by querying the GitHub Releases API |
| `https://github.com/*` | Link to GitHub release pages so you can view release notes or download a release manually |
| `https://*.githubusercontent.com/*` | Download Save Sora update packages from GitHub release assets |

<Info>
  The `https://soravdl.com/*` host permission covers the extension's own domain. No personal data from your Sora library is sent to this domain.
</Info>

<Warning>
  Save Sora does not use any of these host permissions to send your Sora library, auth tokens, or cookies to a developer-owned backend. Network access to Sora domains is used only to fetch your own videos directly from OpenAI's servers into your local browser session.
</Warning>
